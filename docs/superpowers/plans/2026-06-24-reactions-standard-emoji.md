# Reactions standard-emoji + split-error — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `msg.react` accept a built-in standard-emoji set (no per-site seeding) and split the collapsed validation error, without adding a dependency to the main module.

**Architecture:** A committed generated allowlist (`pkg/emoji/standard_emoji_gen.go`) is checked in `Validate` before the custom-store lookup; the generator that reads `kyokomi/emoji` lives in an isolated nested module so the main `go.mod` is untouched. The collapsed error in `reactions.go` is split into invalid (format) vs unknown (not registered).

**Tech Stack:** Go 1.25.11, NATS RPC, MongoDB custom-emoji store, testify. Spec: `docs/superpowers/specs/2026-06-24-reactions-standard-emoji-design.md`. Issue #382.

```go
// Command gen writes pkg/emoji/standard_emoji_gen.go: the built-in reaction
// shortcode allowlist, derived from kyokomi/emoji and filtered to the wire regex.
package main

import (
	"os"
	"regexp"
	"sort"
	"strings"
	"text/template"

	"github.com/kyokomi/emoji/v2"
)

var wire = regexp.MustCompile(`^[a-z0-9_+-]{1,32}$`)

var tmpl = template.Must(template.New("g").Parse(`// Code generated by pkg/emoji/gen; DO NOT EDIT.
package emoji

var standardEmoji = map[string]struct{}{
{{range .}}	{{printf "%q" .}}: {},
{{end}}}
`))

func main() {
	set := map[string]struct{}{}
	for k := range emoji.CodeMap() {
		name := strings.Trim(k, ":")
		if wire.MatchString(name) {
			set[name] = struct{}{}
		}
	}
	names := make([]string, 0, len(set))
	for n := range set {
		names = append(names, n)
	}
	sort.Strings(names)

	f, err := os.Create("../standard_emoji_gen.go")
	if err != nil {
		panic(err)
	}
	defer f.Close()
	if err := tmpl.Execute(f, names); err != nil {
		panic(err)
	}
}
```
