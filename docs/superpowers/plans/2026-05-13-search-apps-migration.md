# search.apps Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the legacy `GET /api/v3/apps` HTTP endpoint into `search-service` as a NATS request/reply RPC at `chat.user.{account}.request.search.apps`, returning apps the caller has subscribed to (filtered by name, optionally by `assistant.enabled`).

**Architecture:** Add a Mongo connection to `search-service` (alongside the existing ES + Valkey backends), expose a new `MongoStore` interface with `SearchAppsByName`, and dispatch from `searchApps` handler. The Mongo aggregation pipeline (`$match` → `$lookup subscriptions` access guard → `$group` → `$lookup rooms` → `$project`) is implemented as a prototype skeleton in this plan; the engineer refines the pipeline body separately.

**Tech Stack:** Go 1.25, NATS request/reply (`pkg/natsrouter`), MongoDB aggregation (driver v2), `pkg/mongoutil.Connect`, `caarlos0/env`, mockgen, testify, testcontainers (`pkg/testutil.MongoDB`).

**Spec:** `docs/superpowers/specs/2026-05-13-search-service-nats-migrations-design.md` (sections 2, 3, 4, 5.1, 6, 7.1, 9, 10, 11, 12 Phase 0+1).

**Branch:** `claude/migrate-search-nats-eHDjs` (already checked out — confirm with `git branch --show-current` before starting).

package main

import (
	"regexp"

	"go.mongodb.org/mongo-driver/v2/bson"
)

// buildSearchAppsPipeline builds the Mongo aggregation pipeline for
// `search.apps`. The current body is a minimal-viable PROTOTYPE that
// supports name regex + optional assistant.enabled filter + $limit; the
// full pipeline per the design spec
// (docs/superpowers/specs/2026-05-13-search-service-nats-migrations-design.md §5.1)
// additionally $lookups subscriptions (the user-scope access guard),
// performs a $group, and $lookups rooms before $project. Extend this
// function as the real pipeline gets authored; the function signature
// is the stable contract.
//
// All metacharacters in query are escaped via regexp.QuoteMeta so
// a caller can't inject regex syntax (ReDoS or pattern injection).
// The match is case-insensitive substring (no anchors).
func buildSearchAppsPipeline(
	query, account string,
	assistantEnabled *bool,
	limit int,
) []bson.M {
	matchStage := bson.M{
		"name": bson.M{
			"$regex":   regexp.QuoteMeta(query),
			"$options": "i",
		},
	}
	if assistantEnabled != nil {
		matchStage["assistant.enabled"] = *assistantEnabled
	}

	// TODO(searchApps-pipeline): replace this prototype body with the
	// full spec'd pipeline:
	//   1. $match (this stage — keep)
	//   2. $lookup against `subscriptions` (the access guard: drop apps
	//      the user `account` has not subscribed to)
	//   3. $group / projection to deduplicate and shape per-app
	//   4. $lookup against `rooms` (decorate with per-user room info if
	//      the response shape requires it; today's spec returns model.App
	//      so room info is consumed inside the pipeline, not projected)
	//   5. $limit (this stage — keep, last)
	//   6. terminal $project matching model.App's bson tags
	_ = account // referenced in the access-guard $lookup once implemented

	return []bson.M{
		{"$match": matchStage},
		{"$limit": limit},
	}
}
```

package main

import (
	"context"
	"fmt"

	"go.mongodb.org/mongo-driver/v2/mongo"

	"github.com/hmchangw/chat/pkg/model"
)

// mongoStore is the Mongo-backed implementation of MongoStore.
// Binds only the `apps` collection in Go; pipelines reference
// `subscriptions` and `rooms` by name in the same database, owned by
// other services (room-service / room-worker).
type mongoStore struct {
	apps *mongo.Collection
}

func newMongoStore(db *mongo.Database) *mongoStore {
	return &mongoStore{apps: db.Collection("apps")}
}

func (s *mongoStore) SearchAppsByName(
	ctx context.Context,
	query, account string,
	assistantEnabled *bool,
	limit int,
) ([]model.App, error) {
	pipeline := buildSearchAppsPipeline(query, account, assistantEnabled, limit)
	cur, err := s.apps.Aggregate(ctx, pipeline)
	if err != nil {
		return nil, fmt.Errorf("aggregate apps: %w", err)
	}
	defer cur.Close(ctx)

	results := make([]model.App, 0)
	if err := cur.All(ctx, &results); err != nil {
		return nil, fmt.Errorf("decode apps: %w", err)
	}
	return results, nil
}
```

func (h *handler) searchApps(c *natsrouter.Context, req model.SearchAppsRequest) (resp *model.SearchAppsResponse, err error) {
	defer observeRequest(metricKindApps, &err)()

	account, rerr := c.Params.Require("account")
	if rerr != nil {
		return nil, rerr
	}

	if err := h.normalizePagination(&req.Size, &req.Offset); err != nil {
		return nil, err
	}

	query := strings.TrimSpace(req.Query)
	if query == "" {
		return nil, natsrouter.ErrBadRequest("query is required")
	}

	ctx, cancel := h.withRequestTimeout(c)
	defer cancel()

	apps, err := h.mongo.SearchAppsByName(ctx, query, account, req.AssistantEnabled, req.Size)
	if err != nil {
		slog.Error("app search backend failed", "account", account, "error", err)
		return nil, natsrouter.ErrInternal("search backend unavailable")
	}

	if apps == nil {
		apps = []model.App{}
	}
	return &model.SearchAppsResponse{Apps: apps}, nil
}

**Subject:** `chat.user.{account}.request.search.apps`

**Auth:** the `{account}` in the subject is the authenticated identity (enforced by the NATS auth callout). The response is scoped to apps the user has subscribed to; results are filtered by the pipeline's `$lookup` against the `subscriptions` collection.

  "size": 25,

|---|---|---|---|

| `offset` | integer | no | Page offset. Default `0`. |

**Response body:**
```json
{
  "apps": [
    {
      "id": "a1",
      "name": "Weather",

      "assistant": { "enabled": true, "name": "weather.bot", "settingsUrl": "..." },
      "sponsors": [{ "name": "...", "phone": "..." }]
    }
  ]
}
```

| Code | Reason |
